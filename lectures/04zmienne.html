Zmienne
Z każdą zmienną związane są:
  • nazwa
  • typ
  • adres
  • wartość
  • zasięg
Nazwa zmiennej:
  • ciąg liter i cyfr rozpoczynający się od litery. Znak podkreślenia _ traktujemy tutaj jako literę
  • małe i wielkie litery są rozróżnialne !
  • nie mogą być słowami kluczowymi języka
  • nazwy mogą być dowolnej długości, przy czym w niektórych implementacjach tylko pewien rozsądny prefiks jest brany pod uwagę
Deklaracje i definicje
  • deklaracja przedstawia kompilatorowi identyfikator - nazwę. Nieformalnie: ta zmienna (funkcja) istnieje w jakimś miejscu i powinna tak wyglądać
  • definicja - nakazuje kompilatorowi: utwórz w tym miejscu zmienną i przydziel pamięć
  • w danym zasięgu: bloku instrukcji, funkcji, pliku, programie może wystąpić tylko jedna definicja zmiennej o danej nazwie
  • definicja może być również deklaracją
Zmienne globalne
Zasięg globalny
  • zmienne dostępne w całym programie
  • jeśli chcemy użyć zmiennej globalnej w innym pliku niż została ona zdefiniowana musimy ją zadeklarować
  • deklaracja "na zewnątrz" wszystkich funkcji
  • dla własnego dobra powinniśmy jawnie wskazać, że zmienna jest zdefiniowana "gdzie indziej": słowo kluczowe extern
  • zmienne globalne są inicjalizowane wartościami zero
Zmienne globalne: przykład
Plik pierwszy.c: 
extern int liczba;  // deklaracja zmiennej globalnej
int main(){
 //...
 printf("%d\n", liczba);
}
Plik drugi.c: 
int liczba = 5;    // definicja zmiennej globalnej 
int funkcja1(){
 //...
}
Kompilacja i łączenie: gcc pierwszy.c drugi.c -o program
Zmienne lokalne
  • lokalne dla pliku
    • zmienne dostępne tylko w danym pliku źródłowym
    • deklarowane przy użyciu słowa kluczowego static
    • deklaracja "na zewnątrz" wszystkich funkcji
    • nie są widoczne w innych plikach, nawet gdy użyjemy extern
  • lokalne funkcji lub bloku instrukcji
    • widoczne od miejsca deklaracji do końca bloku, który otacza deklarację
    • nie są inicjalizowane
    • zwane automatycznymi
Zmienne lokalne: przykład
Zmienne przykrywają swoje wartości: 
// definicja zmiennej globalnej programu        
int liczba = 5;  
int main(){
  // definicja zmiennej lokalnej funkcji
  int liczba = 6;   
  if ( warunek ) {
    // definicja zmiennych lokalnych bloku instrukcji
    int liczba = 7;
    int x = 4;
    printf("%d\n", liczba);
  }
  //printf("%d\n", x);  // błąd!
  printf("%d\n", liczba);
}
Przykrywanie zmiennych może być źródłem trudnych do wykrycia błędów w programie. W Javie zabronione.
Literały
Przykłady literałów różnych typów
  • int i = 9, j = 017, k = 0x12F;
  • char c1 = 'a', c2 = 97;
  • int i = 123456;
  • long l = 12345679L;
  • float f = 1234.567F;
  • double d = 1234.567;
  • long double d = 1234.567L;
  • char szkola[] = "pjwstk";
Literały znaków specjalnych
Literały znaków specjalnych
  • '\n' nowa linia
  • '\t' tabulacja
  • '\0' znak o kodzie 0, null
  • '\\' ukośnik, backslash
  • '\b' odstęp do tyłu backspace
  • '\r' powrót karetki (carriage return)
  • '\f' zmiana formularza (form feed)
  • '\ddd' znak o kodzie ósemkowym
Stałe
  • stałe to zmienne, których nie można zmienić po zainicjalizowaniu
  • definicja stałej musi być połączona z jej inicjalizacją, najczęściej literałem lub ogólniej wartością znaną na etapie kompilacji
  • deklaracja przy użyciu słowa kluczowego const
    const int x = 7;
  • inaczej niż w C++, stałych nie można używać do deklarowania tablic! Zamiast tego dyrektywa preprocesora:
    #define ROZMIAR 20
    char tablica[ROZMIAR];
